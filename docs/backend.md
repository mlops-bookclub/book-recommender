# Backend — FastAPI Setup

## Overview

The FastAPI backend lives under `backend/` and serves as the inference API for the Book Recommender service. It is deployed inside a Docker container and consumed by the Streamlit frontend (see architecture diagram).

## Directory Structure

```
backend/
├── src/
│   ├── __init__.py
│   ├── main.py                  # FastAPI application entry point
│   └── routers/
│       ├── __init__.py
│       ├── health.py            # GET  /health
│       ├── books.py             # GET  /books
│       └── recommend.py        # POST /recommend
├── tests/
│   ├── __init__.py
│   ├── conftest.py              # Shared TestClient fixture
│   ├── test_health.py
│   ├── test_books.py
│   └── test_recommend.py
├── Dockerfile
├── pyproject.toml
├── .python-version              # 3.12
└── uv.lock                      # Auto-generated by uv sync
```

## Dependency Management

Dependencies are managed with [uv](https://docs.astral.sh/uv/). The `uv.lock` file pins all transitive dependencies for reproducible installs.

| Group   | Packages                              |
|---------|---------------------------------------|
| Runtime | `fastapi`, `uvicorn[standard]`, `pydantic` |
| Dev     | `pytest`, `httpx`, `pytest-asyncio`   |

## API Endpoints

### `GET /health`

Liveness check. Returns `200 OK` when the service is running.

**Response**
```json
{ "status": "ok" }
```

---

### `GET /books`

Returns the list of available books. Currently returns an empty list — will be populated once the data pipeline (DVC) is connected.

**Response**
```json
{ "books": [] }
```

---

### `POST /recommend`

Returns book recommendations for a given title.

> **Note:** Currently returns stub data. Real model inference will be wired in [issue #9](https://github.com/mlops-bookclub/book-recommender/issues/9).

**Request body**
```json
{
  "book_title": "The Hunger Games",
  "n_recommendations": 5
}
```

| Field              | Type    | Required | Default | Constraints |
|--------------------|---------|----------|---------|-------------|
| `book_title`       | string  | yes      | —       | —           |
| `n_recommendations`| integer | no       | `5`     | 1 – 20      |

**Response**
```json
{
  "recommendations": [
    { "title": "Catching Fire", "author": "Suzanne Collins", "score": 0.97 },
    { "title": "Mockingjay",    "author": "Suzanne Collins", "score": 0.95 }
  ]
}
```

Interactive docs are available at `http://localhost:8000/docs` (Swagger UI) when the server is running.

## Running Locally

```bash
cd app/backend

# Install dependencies (first time or after pyproject.toml changes)
uv sync

# Start the development server with auto-reload
uv run uvicorn src.main:app --reload
```

The API will be available at `http://localhost:8000`.

## Running Tests

```bash
cd app/backend
uv run python -m pytest tests/ -v
```

Expected output: **6 passed**.

## Docker

### Build

```bash
cd app/backend
docker build -t book-recommender-backend .
```

### Run

```bash
docker run -p 8000:8000 book-recommender-backend
```

The Dockerfile uses `python:3.12-slim`, installs `uv`, copies only `pyproject.toml` and `uv.lock` first (for layer caching), then runs `uv sync --frozen --no-dev` before copying the application source.
